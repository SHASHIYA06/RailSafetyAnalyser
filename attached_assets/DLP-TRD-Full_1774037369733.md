## 1. SYSTEM ARCHITECTURE OVERVIEW

### Architecture Layers

```
┌─────────────────────────────────────────────────────────┐
│              Presentation Layer (Frontend)              │
│  React Web App + Progressive Web App (Mobile)           │
├─────────────────────────────────────────────────────────┤
│              API Gateway & Load Balancer                │
│  (Kong/AWS API Gateway, SSL/TLS Termination)           │
├─────────────────────────────────────────────────────────┤
│              Application Layer (Backend)                │
│  Microservices: Auth, Inventory, Transaction, Reports   │
│  (Node.js/Express or Python/FastAPI)                    │
├─────────────────────────────────────────────────────────┤
│              Data Access Layer                          │
│  ORM (Sequelize/SQLAlchemy), Query Optimization         │
├─────────────────────────────────────────────────────────┤
│              Data Layer                                 │
│  PostgreSQL (Primary), Redis (Cache), Elasticsearch     │
├─────────────────────────────────────────────────────────┤
│      External System Integration Layer                  │
│  NCR, Job Card, Gate Post, Finance, Notification       │
└─────────────────────────────────────────────────────────┘
```

### Component Interaction Diagram

```
Frontend (React)
    ↓
API Gateway (Authorization, Rate Limiting)
    ↓
Microservices
├─ Auth Service (JWT, User Management)
├─ Inventory Service (Item CRUD, Stock Tracking)
├─ Transaction Service (Issue, Receipt, Return)
├─ Search Service (Elasticsearch, Full-Text Search)
├─ Report Service (Generate, Export, Schedule)
├─ Integration Service (NCR, Job Card, Gate Post)
└─ Notification Service (Email, SMS, Dashboard)
    ↓
Data Layer
├─ PostgreSQL (Persistent Data)
├─ Redis (Caching, Session Management)
└─ Elasticsearch (Search Indexing)
    ↓
External Systems
├─ NCR System (Webhooks)
├─ Job Card System (API)
├─ Gate Post System (Barcode Scanning)
└─ Finance System (ERP Integration)
```

## 3. DATABASE DESIGN

### Database Schema Overview

**Core Tables:**
1. `users` - System users
2. `tbl_items` - Item master
3. `tbl_stock_levels` - Current stock at each location
4. `tbl_vendors` - Vendor master
5. `tbl_transactions` - All inventory transactions
6. `tbl_audit_log` - Audit trail
7. `tbl_alerts` - Alert tracking
8. `tbl_integrations` - External system logs

### Table 1: users

```sql
CREATE TABLE users (
    user_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    first_name VARCHAR(100) NOT NULL,
    last_name VARCHAR(100) NOT NULL,
    phone VARCHAR(20),
    role VARCHAR(50) NOT NULL, -- STORE_KEEPER, SUPERVISOR, OP_MANAGER, DEPOT_MANAGER, ADMIN
    location_id INT REFERENCES tbl_locations(location_id),
    is_active BOOLEAN DEFAULT true,
    last_login TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    created_by UUID,
    updated_by UUID,

    INDEX idx_email (email),
    INDEX idx_role (role),
    INDEX idx_is_active (is_active)
);
```

### Table 2: tbl_items (Item Master)

```sql
CREATE TABLE tbl_items (
    item_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    part_number VARCHAR(50) UNIQUE NOT NULL,
    item_description VARCHAR(255) NOT NULL,
    category VARCHAR(100) NOT NULL, -- Brake, Traction, etc.
    system_type VARCHAR(100) NOT NULL, -- Brake System, Traction Motor, etc.
    vendor_id UUID REFERENCES tbl_vendors(vendor_id),
    unit_of_measure VARCHAR(20) NOT NULL,
    recommended_qty INT DEFAULT 10,
    reorder_level INT DEFAULT 5,
    critical_flag BOOLEAN DEFAULT false,
    is_active BOOLEAN DEFAULT true,
    expiry_tracking BOOLEAN DEFAULT false,
    notes TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    created_by UUID,

    UNIQUE(part_number),
    INDEX idx_category (category),
    INDEX idx_system_type (system_type),
    INDEX idx_vendor_id (vendor_id),
    INDEX idx_critical_flag (critical_flag),
    FULLTEXT INDEX idx_search (part_number, item_description)
);
```

### Table 3: tbl_stock_levels (Current Stock)

```sql
CREATE TABLE tbl_stock_levels (
    stock_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    item_id UUID NOT NULL REFERENCES tbl_items(item_id),
    location_id INT NOT NULL REFERENCES tbl_locations(location_id),
    received_qty INT DEFAULT 0,
    consumed_qty INT DEFAULT 0,
    reserved_qty INT DEFAULT 0,
    damaged_qty INT DEFAULT 0,
    available_qty INT GENERATED ALWAYS AS 
        (received_qty - consumed_qty - reserved_qty - damaged_qty) STORED,
    batch_number VARCHAR(50),
    serial_number VARCHAR(100),
    expiry_date DATE,
    last_updated TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_by UUID,

    UNIQUE(item_id, location_id),
    INDEX idx_item_id (item_id),
    INDEX idx_location_id (location_id),
    INDEX idx_available_qty (available_qty),
    INDEX idx_expiry_date (expiry_date)
);
```

### Table 4: tbl_vendors (Vendor Master)

```sql
CREATE TABLE tbl_vendors (
    vendor_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    vendor_name VARCHAR(100) UNIQUE NOT NULL,
    vendor_code VARCHAR(20) UNIQUE NOT NULL,
    contact_person VARCHAR(100),
    email VARCHAR(100),
    phone VARCHAR(20),
    address TEXT,
    country VARCHAR(50),
    payment_terms VARCHAR(50),
    delivery_days INT,
    quality_rating VARCHAR(50), -- Excellent, Good, Average, Poor
    is_active BOOLEAN DEFAULT true,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,

    INDEX idx_vendor_name (vendor_name),
    INDEX idx_vendor_code (vendor_code),
    INDEX idx_is_active (is_active)
);
```

### Table 5: tbl_transactions (All Inventory Movements)

```sql
CREATE TABLE tbl_transactions (
    transaction_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    transaction_type VARCHAR(50) NOT NULL, -- RECEIPT, ISSUE, RETURN, TRANSFER, ADJUSTMENT, WRITE_OFF
    item_id UUID NOT NULL REFERENCES tbl_items(item_id),
    quantity INT NOT NULL,
    from_location_id INT REFERENCES tbl_locations(location_id),
    to_location_id INT REFERENCES tbl_locations(location_id),
    reference_type VARCHAR(50), -- NCR, JOB_CARD, PO, OTHER
    reference_id VARCHAR(50),
    transaction_date TIMESTAMP NOT NULL,
    transaction_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    initiated_by UUID NOT NULL REFERENCES users(user_id),
    approved_by UUID REFERENCES users(user_id),
    approval_date TIMESTAMP,
    remarks TEXT,
    batch_number VARCHAR(50),
    serial_number VARCHAR(100),
    status VARCHAR(50) DEFAULT 'COMPLETED', -- PENDING, COMPLETED, CANCELLED
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,

    INDEX idx_transaction_type (transaction_type),
    INDEX idx_item_id (item_id),
    INDEX idx_reference_id (reference_id),
    INDEX idx_transaction_date (transaction_date),
    INDEX idx_initiated_by (initiated_by),
    INDEX idx_status (status)
);
```

### Table 6: tbl_audit_log (Complete Audit Trail)

```sql
CREATE TABLE tbl_audit_log (
    audit_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(user_id),
    action VARCHAR(50) NOT NULL, -- CREATE, READ, UPDATE, DELETE, EXPORT, DOWNLOAD
    entity_type VARCHAR(50) NOT NULL, -- ITEM, TRANSACTION, VENDOR, USER, etc.
    entity_id VARCHAR(100),
    old_values JSONB,
    new_values JSONB,
    ip_address VARCHAR(45),
    user_agent TEXT,
    timestamp TIMESTAMP DEFAULT CURRENT_TIMESTAMP,

    INDEX idx_user_id (user_id),
    INDEX idx_action (action),
    INDEX idx_entity_type (entity_type),
    INDEX idx_timestamp (timestamp)
);
```

### Table 7: tbl_alerts (Alert Management)

```sql
CREATE TABLE tbl_alerts (
    alert_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    alert_type VARCHAR(50) NOT NULL, -- LOW_STOCK, CRITICAL, OVERSTOCK, EXPIRY, DISCREPANCY
    item_id UUID REFERENCES tbl_items(item_id),
    severity VARCHAR(20) NOT NULL, -- CRITICAL, WARNING, INFO
    alert_message TEXT NOT NULL,
    alert_details JSONB,
    triggered_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    resolved_at TIMESTAMP,
    resolved_by UUID REFERENCES users(user_id),
    notification_sent BOOLEAN DEFAULT true,
    notification_channels TEXT, -- EMAIL, SMS, DASHBOARD, comma-separated
    is_active BOOLEAN DEFAULT true,

    INDEX idx_alert_type (alert_type),
    INDEX idx_item_id (item_id),
    INDEX idx_severity (severity),
    INDEX idx_triggered_at (triggered_at),
    INDEX idx_is_active (is_active)
);
```

### Table 8: tbl_integrations (External System Logs)

```sql
CREATE TABLE tbl_integrations (
    integration_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    system_name VARCHAR(50) NOT NULL, -- NCR, JOB_CARD, GATE_POST, FINANCE
    event_type VARCHAR(100) NOT NULL,
    payload JSONB,
    response JSONB,
    status VARCHAR(20) NOT NULL, -- SUCCESS, FAILED, PENDING
    error_message TEXT,
    retry_count INT DEFAULT 0,
    last_retry TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,

    INDEX idx_system_name (system_name),
    INDEX idx_event_type (event_type),
    INDEX idx_status (status),
    INDEX idx_created_at (created_at)
);
```

### Additional Tables

```sql
-- Locations
CREATE TABLE tbl_locations (
    location_id SERIAL PRIMARY KEY,
    location_name VARCHAR(100) UNIQUE NOT NULL,
    location_code VARCHAR(20) UNIQUE NOT NULL,
    description TEXT,
    is_active BOOLEAN DEFAULT true,

    INDEX idx_location_name (location_name)
);

-- Reports (Store scheduled reports)
CREATE TABLE tbl_reports (
    report_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    report_name VARCHAR(100) NOT NULL,
    report_type VARCHAR(50) NOT NULL,
    query_config JSONB,
    schedule VARCHAR(50), -- DAILY, WEEKLY, MONTHLY, ONE_TIME
    next_run TIMESTAMP,
    created_by UUID REFERENCES users(user_id),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    is_active BOOLEAN DEFAULT true
);

-- Report Executions
CREATE TABLE tbl_report_executions (
    execution_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    report_id UUID REFERENCES tbl_reports(report_id),
    executed_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    executed_by UUID REFERENCES users(user_id),
    row_count INT,
    file_path VARCHAR(255),
    file_format VARCHAR(20), -- CSV, PDF, XLSX, JSON
    status VARCHAR(20) -- SUCCESS, FAILED
);
```

### Indexing Strategy

**Critical Indexes for Performance:**
- `tbl_items` (part_number, category, system_type, vendor_id)
- `tbl_stock_levels` (item_id, location_id, expiry_date)
- `tbl_transactions` (item_id, transaction_type, transaction_date)
- `tbl_audit_log` (user_id, entity_type, timestamp)

**Full-Text Search Indexes:**
- `tbl_items` (part_number, item_description)

**Composite Indexes:**
- `tbl_stock_levels` (item_id, location_id) - UNIQUE
- `tbl_transactions` (item_id, from_location_id, to_location_id, transaction_date)

## 6. BACKEND SPECIFICATIONS

### API Rate Limiting

```
Per User: 1000 requests/hour
Per IP: 5000 requests/hour
Burst: 100 requests/second
Response 429: When exceeded
```

### Authentication & Authorization

**JWT Token Structure:**
```
Header: {
  "alg": "HS256",
  "typ": "JWT"
}
Payload: {
  "sub": "user_id",
  "email": "user@kmrcl.gov.in",
  "role": "STORE_KEEPER",
  "iat": 1710854400,
  "exp": 1710858000
}
```

**Token Expiry:**
- Access Token: 1 hour
- Refresh Token: 7 days
- Max Session: 30 days (must re-login)

### Error Handling

**Standard Error Response:**
```json
{
  "error": "Error code",
  "message": "Human readable message",
  "details": {
    "field": "error details"
  },
  "timestamp": "2026-03-19T10:30:00Z",
  "request_id": "uuid"
}
```

**HTTP Status Codes:**
- 200: Success
- 201: Created
- 202: Accepted (async)
- 400: Bad Request
- 401: Unauthorized
- 403: Forbidden
- 404: Not Found
- 409: Conflict
- 422: Validation Error
- 429: Too Many Requests
- 500: Server Error
- 503: Service Unavailable

### Caching Strategy

**Redis Cache Keys:**
```
user:{user_id} - User session (TTL: 1 hour)
item:{item_id} - Item details (TTL: 24 hours)
stock:{item_id}:{location_id} - Stock levels (TTL: 5 minutes)
alerts:active - Active alerts (TTL: 1 minute)
report:{report_id} - Cached reports (TTL: 6 hours)
```

**Cache Invalidation:**
- On item update: Invalidate item:{item_id}
- On stock change: Invalidate stock:{item_id}:{location_id}
- On alert: Invalidate alerts:active
- Manual purge on demand

### Background Jobs

**Async Task Queue (Bull/RQ):**

1. **Stock Alert Check** (every 5 minutes)
   - Check items below reorder level
   - Send notifications
   - Create alerts

2. **Expiry Alerts** (daily at 2 AM)
   - Check items expiring soon
   - Send notifications
   - Create alerts

3. **Report Generation** (scheduled)
   - Generate scheduled reports
   - Send via email
   - Archive to S3

4. **Backup** (daily at 11 PM)
   - Database backup
   - Upload to S3
   - Verify backup

5. **Integration Retry** (every 30 minutes)
   - Retry failed webhook calls
   - Sync with NCR, Job Card, Gate Post
   - Log results

6. **Audit Log Archive** (monthly)
   - Archive old audit logs to cold storage
   - Compress and encrypt

## 8. SECURITY & AUTHENTICATION

### Security Measures

**Data Security:**
- Encryption at REST: AES-256
- Encryption in TRANSIT: TLS 1.3
- Password Hashing: bcrypt (10 rounds minimum)
- API Keys: Rotated every 90 days

**Access Control:**
- Role-Based Access Control (RBAC)
- Multi-level approval for sensitive operations
- IP whitelisting for admin operations
- Session timeout: 30 minutes inactivity

**Audit & Compliance:**
- All transactions logged with user ID
- Change history maintained
- Compliance reports generated on demand
- Regular security audits (quarterly)

### Authentication Flow

```
1. User enters email + password
   ↓
2. Server verifies credentials
   - Query user from database
   - Compare bcrypt hash
   - If valid, proceed to step 3
   - If invalid, increment login attempt counter
   - After 5 failures in 15 min, lock account
   ↓
3. Generate JWT tokens
   - Create access token (1 hour)
   - Create refresh token (7 days)
   - Store refresh token hash in database
   ↓
4. Return tokens to client
   - Client stores in secure session storage
   - Client sets Authorization header on subsequent requests
   ↓
5. For subsequent requests
   - Client sends: Authorization: Bearer [access_token]
   - Server verifies signature + expiry
   - If valid, process request
   - If expired, client sends refresh token
   ↓
6. Token refresh
   - Client sends: POST /auth/refresh with refresh_token
   - Server verifies refresh token
   - Generate new access token (1 hour)
   - Return to client
```
